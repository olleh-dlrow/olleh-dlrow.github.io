文档模板

目的

总结一些常见的优质API文档或开发文档的模板，用于编写文档的借鉴

Microsoft C++ 标准库

https://docs.microsoft.com/zh-cn/cpp/standard-library/cpp-standard-library-reference?view=msvc-170

结构

  • 标准库参考：总介绍标准库的作用，然后引导至标准库概述

  • 标准库概述：引导至标准库的使用方法和标准库的API文档

    • 使用C++库标头
    • C++库约定
    • C++程序启动和终止
    • …

  • C++标准库头文件：核心的API文档，使用文件作为子结构划分，还包含一页目录，用于引导至各个文件的API文档

    • C++标准库头文件：按类别划分的标头，包含各个API文档的索引表，第1列为超链接，第2列为简短的介绍

      文件名称 简短介绍
      file1 some description
      file2 somedescription
      … …

    • file1: 某个头文件，包含2个部分，该文件的API目录和API功能、内容介绍

      • file1目录：按照API划分，包含各个API的索引表，也是1列超链接，1列简短介绍

        API名称 简短介绍
        API1 some description
        API2 some description
        … …

      • file1API文档：包含API的名称，功能，参数，返回值，备注，示例等

        二级标题 API1

        功能，接口代码等

        二级标题 参数

        arg1

        some description

        arg2

        some description

        …

        二级标题 返回值

        介绍返回值

        二级标题 备注

        包含一些细节和注意事项

        二级标题 示例

        代码示例及其输出

    • file2

    • …

小结

API文档的核心就是描述其功能，参数，返回值以及注意事项，不需要更多的实现细节，和介绍某个工具的使用方法类似，目的在于让用户知道怎么用，在哪儿用，让用户尽快上手进行后续开发

文档的结构应尽量少一些嵌套，这样能使树形结构不至于太复杂，同时，很多情况下一个API能够根据多个分类标准进行分类，比如根据命名空间分类、根据模块分类、根据文件分类、根据所属类分类等，这些分类都可以通过目录表的方式展现出来，每个目录表放在文档的开头，然后目录项索引到API的介绍内容
